## 0. 과정 한 줄 정의
AI 에이전트로 취약점을 **선제 탐지·자동 패치·검증**하는 "창과 방패" 운영 역량과, 보안을 개발자 수준으로 내려보내는 **Secure-by-default 전파(Enablement)** 역량을 기른다.

## 1. 대상 학습자 & 진입 역량
- **대상** : Security 트랙. **진입 역량** : SE-01, 시큐어 코딩, CI/CD 보안 기초.
- **왜 Security 트랙 전용인가** : 자동 탐지·패치 운영과 가드레일·SBOM 설계는 보안 전문 영역. 다만 *전파 대상*은 전 개발자(공통)이므로 "**개발자 수준의 보안 교육**"(보안 거버넌스 리더(F))을 설계·전파하는 것까지 포함.

## 2. 교수설계 근거
- **적용 방법론** : Backward Design · Bloom · Gagné · Merrill · CBE.
- **설계 데이터 근거**
  - (내부) "**창과 방패** — AI로 자동 패치 + 동시에 상대보다 먼저 선제 리스트업"(보안 거버넌스 리더(F)) / "이제는 **해커가 아니어도 AI로 해커 수준** 취약점 발견" / "하루에도 **10배**의 보안 이슈, 자사는 **80%+ 오픈소스**" / "이건 **개발자 수준으로 내려와야**, 개발자 수준 교육 필요"(보안 거버넌스 리더(F)).
  - (외부) Google **Big Sleep**(자율 취약점 탐지) / **XBOW**(HackerOne 1위) / 개발자 주도 위협모델링(Security Boulevard) / OWASP secure coding.

## 3. Stage 1 — 기대 성취
- **핵심 질문** : 공격자가 AI를 쓰는 시대에 방어는 어떻게 같은 무기를 드는가? 보안을 어떻게 개발자 수준으로 내려보내는가?
- **전이 목표** : AI 기반 **자동 탐지·패치·리그레션**을 운영하고, **Secure-by-default 가드레일·교육**을 조직에 전파한다.
- **이해** : ① 창과 방패는 둘 다 필요 ② 오픈소스 비중↑ → 공급망·SBOM 관리 ③ 보안은 보안팀+개발팀 공동.
- **학습목표(Bloom)**
  - LO1 (적용) AI 에이전트로 취약점을 **선제 탐지**한다.
  - LO2 (분석) 자동 패치의 부작용·**리그레션**을 분석한다.
  - LO3 (평가) **SBOM/ML-BOM·공급망** 위험을 평가한다.
  - LO4 (창안) **가드레일 + 개발자용 Secure-by-default 교육/체크**를 설계·전파한다.

## 4. Stage 2 — 평가 증거 / DoD
- **수행과제** : 현업 리포지토리에 **AI 자동 취약점 탐지→패치→리그레션 검증** 파이프라인을 구성하고, 개발자용 **secure-by-default 가드레일/가이드**를 제작·배포.
- **루브릭** : ① 탐지 적중률 ② 패치 안전성(리그레션) ③ SBOM/공급망 커버리지 ④ 전파물(교육/가드레일) 실용성.
- **이수 DoD** : 자동 탐지·패치가 **실제 취약점 1건 이상 처리** + 리그레션 미발생 + 개발자 가드레일 배포.
- **나노디그리 이수 조건** : 수행과제 + 파이프라인 산출물 + 전파물(가드레일/가이드).

## 5. Stage 3 — 학습 계획 (모듈)
- **M1. AI-for-Security 개관** — 창과 방패, 자율 헌터(Big Sleep/XBOW). *(주의환기)*
- **M2. 자동 탐지·패치·리그레션** — 파이프라인 구성, 부작용 관리. *(시연→적용)*
- **M3. 공급망·SBOM/ML-BOM** — 오픈소스 80%+ 대응, 가드레일. *(적용)*
- **M4. Secure-by-default 전파** — 개발자 수준 교육/체크 설계·배포. *(통합 / Enablement)*

## 6. 도구 & 자원
- **도구** : AI 취약점 스캐너/패치 에이전트, SBOM/ML-BOM 도구, 가드레일 프레임워크.
- **자원** : Google Big Sleep, XBOW/HackerOne, OWASP secure coding, 개발자 주도 위협모델링.
