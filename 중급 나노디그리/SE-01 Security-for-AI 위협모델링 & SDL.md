## 0. 과정 한 줄 정의
AI 탑재 제품의 **다른 공격 표면(프롬프트 인젝션·RAG 포이즈닝·적대적 공격)**을 모델링하고, 위협모델링·SDL·보안 요구사항을 **AI용으로 재정의**하는 역량을 기른다.

## 1. 대상 학습자 & 진입 역량
- **대상** : Security 트랙 지망(보안으로 심화하려는 개발자 포함).
- **진입 역량** : CC-01·CC-02, 시큐어 코딩·기본 위협 인식(초급 보안).
- **왜 Security 트랙 전용인가** : "AI 제품은 **공격 방법론이 다르다**"(보안 거버넌스 리더(F)) → 위협모델링·SDL의 *AI용 재정의*는 보안 전문가의 심화 영역. (단, secure-by-default 베이스라인은 초급/공통.)

## 2. 교수설계 근거
- **적용 방법론** : Backward Design · Bloom · Gagné · Merrill · CBE.
- **설계 데이터 근거**
  - (내부) "앞으로 **AI 기능 없는 자사 제품은 없다 → Security-for-AI가 디폴트**"(보안 거버넌스 리더(F)) / "공격 방법론이 다름 — **LLM 입력단 인젝션, 출력단 가로채기, RAG DB 포이즈닝, 적대적 공격** → 위협모델링·침투테스트를 AI용으로 다시" / "**Security·SDET는 개발 전 과정에 걸쳐** 있다 → 게이팅이 워크플로우 전반에 내장"(보안 책임(B)).
  - (외부) Simon Willison **"치명적 3요소(프라이빗 데이터 + 비신뢰 콘텐츠 + 외부 통신)"** / **OWASP Top 10 for LLM Apps + Agentic Apps(2026)** / **MITRE ATLAS** / AI용 **STRIDE**.

## 3. Stage 1 — 기대 성취
- **핵심 질문** : AI 제품의 공격 표면은 전통 SW와 무엇이 다른가? 어디에 게이트를 두어야 하는가?
- **전이 목표** : 현업 AI 기능의 위협을 모델링하고 **SDL/보안 요구사항을 워크플로우에 내장**한다.
- **이해** : ① 입력·출력·데이터(RAG)·모델이 모두 공격면 ② 보안은 전 과정에 걸침 ③ 게이트는 워크플로우 전반에.
- **학습목표(Bloom)**
  - LO1 (적용) AI 기능에 **STRIDE/ATLAS** 위협모델링을 적용한다.
  - LO2 (분석) 인젝션·포이즈닝·적대적 공격·**치명적 3요소** 노출을 분석한다.
  - LO3 (평가) OWASP LLM/Agentic Top 10 기준으로 위험을 **평가**한다.
  - LO4 (창안) **AI용 SDL·보안 요구사항·게이트**를 설계한다.

## 4. Stage 2 — 평가 증거 / DoD
- **수행과제** : 현업 AI 기능 1건의 **위협모델(STRIDE/ATLAS) + OWASP 매핑 + SDL 게이트**를 작성하고, 인젝션/포이즈닝 시나리오를 시연·완화.
- **루브릭** : ① 공격면 도출 완전성 ② 위협모델 타당성 ③ OWASP 매핑 정확성 ④ 게이트/완화 실효성.
- **이수 DoD** : 치명적 3요소 노출 식별 + **프롬프트 인젝션 1건 이상 재현·완화** + SDL 게이트 정의.
- **나노디그리 이수 조건** : 수행과제 + 위협모델 문서 + 완화 시연 로그.

## 5. Stage 3 — 학습 계획 (모듈)
- **M1. AI 공격 표면** — 인젝션·출력 가로채기·RAG 포이즈닝·적대적 공격, 치명적 3요소. *(주의환기)*
- **M2. 위협모델링 재정의** — STRIDE·MITRE ATLAS의 AI 적용. *(시연→적용)*
- **M3. OWASP LLM/Agentic Top 10** — 기준·매핑·우선순위. *(적용)*
- **M4. AI용 SDL & 게이트** — 보안 요구사항, 워크플로우 내장. *(통합)*
