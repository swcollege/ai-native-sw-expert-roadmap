# CC-01 컨텍스트·하네스 엔지니어링 & 토큰 효율 — 중급 나노디그리

> 트랙 : **공통(전 트랙)** · 분류 : **공통 필수** · 난이도 : 중급(L3 · 개인 단위)
> 학습시간 : 32h (8주 × 4h, 자기주도 + 실습) · 배지 : *Context & Harness Practitioner* · 선수 : 초급 AI 역량 인증
> 작성 : 2026-06-15 / SW 공학연구소(SW College) · 근거 : `1차 중간보고_v2.md` + 외부 현황(2026)

## 0. 과정 한 줄 정의
AI 에이전트가 "한 번에 제대로" 일하도록 **컨텍스트를 큐레이션하고, 토큰을 예산처럼 운영하며, 스스로 교정되는 피드백 루프(하네스)를 구성**하는 전 개발자 공통 역량을 기른다.

## 1. 대상 학습자 & 진입 역량 (Entry)
- **대상** : 초급(AI 역량 인증) 이수자 전원. 트랙(Architect/SDET/Security) 불문.
- **진입 역량** : 기본 프롬프트 작성, 에이전트 도구(Claude Code/Cursor 등) 사용 경험, 도커 기초.
- **비대상** : 없음 — 본 과정은 모든 중급 학습자의 **선수 공통과정**.

## 2. 교수설계 근거 (Instructional Design Rationale)
- **적용 방법론** : Backward Design(UbD) 3단계 · Bloom(개정) 학습목표 · Gagné 9 Events(모듈 구조) · Merrill 제1원리(과제중심) · CBE 배지.
- **설계 데이터 근거**
  - (내부) "하네스를 한 번 구축하면 **하루 만에 토큰을 다 쓴다**"(임베디드 리더(C)) / "전체 코드를 통째로 넣으니 토큰이 며칠 만에 죽는다 → **설계서+에러 부분+주변 코드만**"(거버넌스 책임(D)) / "**자연어·추론 과다가 토큰을 가장 많이 먹는다 → 커맨드 바이 커맨드로 처리(processing)화**"(보안 책임(B)) / "앱이 한 컨텍스트에 담길 크기로 규격화 → **도커 기본 지식**"(임베디드 리더(C)).
  - (외부) Fowler/Böckeler **"Agent = Model + Harness"**, Guides(피드포워드)+Sensors(피드백) / Karpathy "컨텍스트 윈도우를 *딱 맞는 정보*로 채우는 기술" / FinOps-for-AI *cost-per-token·cost-per-successful-interaction* / Anthropic "Effective Context Engineering".
- **왜 공통인가** : "AI 역량은 개발자 본연의 역량"(보안 책임(B))·"하네스/컨텍스트는 4개 역할 공통 1순위"(임베디드 리더(C)) → 특정 전문가가 아닌 **전원의 기본기**.

## 3. Stage 1 — 기대 성취 (Desired Results)
- **핵심 질문** : 같은 모델로 왜 누구는 한 번에, 누구는 며칠치 토큰을 태우는가? 무엇을 컨텍스트에 넣고 무엇을 빼야 하는가?
- **전이 목표(Transfer)** : 현업에서 **토큰 예산 내에서** 에이전트가 의도대로 동작하도록 컨텍스트·하네스를 직접 설계·운영한다.
- **이해(Understandings)** : ① 컨텍스트는 '많이'가 아니라 '딱 맞게' ② 추론(자연어)보다 커맨드化가 싸고 정확 ③ 실수는 피드백 루프로 *구조적으로* 차단.
- **학습목표(Bloom)**
  - LO1 (적용) AGENTS.md/스킬(.md)·제약문서로 **Guides**를 구성한다.
  - LO2 (분석) 토큰 소비를 프로파일링해 낭비 지점(자연어 과다·전체 투입)을 **진단**한다.
  - LO3 (평가) 동일 과제를 챗 모드 vs 커맨드化로 비교해 비용·정확도를 **평가**한다.
  - LO4 (창안) 재시도·오류복구가 내장된 피드백 루프(**Sensors**)를 설계한다.

## 4. Stage 2 — 평가 증거 / DoD (Assessment Evidence)
- **수행과제(Performance Task)** : 본인 현업 과제 1건을 골라 **컨텍스트 패키지(설계서+관련 코드+제약) + 커맨드化 워크플로우 + 피드백 루프**를 구축하고 before/after 토큰·성공률을 측정·보고.
- **루브릭(요약)** : ① 컨텍스트 큐레이션(불필요 제거) ② 토큰 효율(개선율) ③ 하네스 견고성(재시도·게이트) ④ 재현성/문서화.
- **이수 DoD** : 동일 과제에서 **토큰 ≥40% 절감 또는 성공률 유의 향상** + 피드백 루프가 1회 이상 자동 교정을 시연.
- **나노디그리 이수 조건** : 수행과제 통과 + 모듈 퀴즈 + 코드/문서 산출물 제출.

## 5. Stage 3 — 학습 계획 (Learning Plan / 모듈)
- **M1. 컨텍스트 엔지니어링 기초** — 컨텍스트 윈도우, 큐레이션(넣을 것/뺄 것), AGENTS.md·CLAUDE.md·스킬(.md). *(Gagné: 주의환기·선행조직자 / Merrill: 시연)*
- **M2. 토큰 경제학 & 커맨드化** — 토큰 프로파일링, 추론 최소화, 챗→처리(processing) 전환, FinOps 지표. *(분석 실습)*
- **M3. 하네스 구성** — Guides(피드포워드)+Sensors(피드백), 재시도·오류복구, 실행환경(도커). *(적용)*
- **M4. 통합 프로젝트** — 현업 과제 적용, 측정·보고. *(Merrill: 통합)*
