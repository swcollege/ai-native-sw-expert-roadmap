# SD-02 테스트 하네스 · CT 피드백 루프 & 게이팅 자동화 — 중급 나노디그리

> 트랙 : **SDET** · 분류 : **트랙 전용** · 난이도 : 중급(L3)
> 학습시간 : 40h (10주 × 4h) · 배지 : *Test Harness & Gating Engineer* · 선수 : CC-01·CC-02 + SD-01 권장
> 작성 : 2026-06-15 / SW 공학연구소(SW College) · 근거 : `1차 중간보고_v2.md` + 외부 현황(2026)

## 0. 과정 한 줄 정의
"틀리면 또 돌리는" **CT 피드백 루프**가 잘 돌도록 테스트 전략·하네스를 설계하고, 워크플로우 산출물의 **게이팅을 평가하는 자동화 에이전트**를 구축하는 역량을 기른다.

## 1. 대상 학습자 & 진입 역량
- **대상** : SDET 트랙. **진입 역량** : SD-01, CI/CD·테스트 자동화 기초.
- **왜 SDET 트랙 전용인가** : "개인이 CICD를 스스로 구축할 필요는 없다(어시스턴트가 해줌). **제품·피처 레벨의 CICD/CT**가 중요"(임베디드 리더(C)) → 제품 레벨 검증 하네스는 SDET의 *전문 영역*.

## 2. 교수설계 근거
- **적용 방법론** : Backward Design · Bloom · Gagné · Merrill · CBE.
- **설계 데이터 근거**
  - (내부) "검증의 핵심은 **CICD/CT 자동화와 피드백 루프** — 틀리면 또 돌리고, 또 돌리는 루프가 잘 만들어져야 하네스가 돈다"(임베디드 리더(C)) / "**뮤테이션 테스트**도 예전엔 말도 안 됐는데 이제 수분이면 끝난다"(로봇제어 책임(H)) / "워크플로우 간 산출물의 **게이팅 수준을 평가하는 AI 에이전트**를 만들 수 있는 사람"(PO 리더(A)) / ISO 29119 등 표준 기반 실무.
  - (외부) **Agentic Testing**(의도 기반, Selenium/Playwright 위) / Fowler **Sensors**(tests as feedback)·mutation/structural testing 재조명 / "**자동화를 만드는 사람은 자동화할 수 없다**".

## 3. Stage 1 — 기대 성취
- **핵심 질문** : 무엇을 어떤 기준으로 테스트할 것인가(테스트 디자인)? 산출물의 게이트를 누가/무엇이 평가하는가?
- **전이 목표** : 제품·피처 레벨에서 **CT 피드백 루프 + 게이팅 자동화 에이전트**를 구축해 하네스가 자가 교정되게 한다.
- **이해** : ① 테스트 *디자인*(어떤 기준)이 핵심 ② 피드백 루프 없는 하네스는 안 돈다 ③ 게이트는 자동화하되 기준은 사람이.
- **학습목표(Bloom)**
  - LO1 (적용) 테스트 전략·레벨·환경(테스트 하네스)을 **설계**한다.
  - LO2 (분석) CT 파이프라인의 **피드백 루프 단절·지연**을 진단한다.
  - LO3 (평가) 게이트 기준으로 산출물 통과/차단을 **자동 평가**한다.
  - LO4 (창안) **게이팅 자동화 에이전트**와 뮤테이션/구조 테스트를 구성한다.

## 4. Stage 2 — 평가 증거 / DoD
- **수행과제** : 현업 피처에 **CT 피드백 루프(실패→재시도→수정) + 게이팅 자동화 에이전트**를 구축하고, 뮤테이션/구조 테스트로 견고성을 입증.
- **루브릭** : ① 테스트 디자인 적절성 ② 피드백 루프 자동성 ③ 게이트 정확도(오탐/미탐) ④ 표준(ISO 29119) 정합.
- **이수 DoD** : 게이팅 에이전트가 불량 산출물 **자동 차단** + 피드백 루프 자가 교정 1회 이상 + 뮤테이션 점수 보고.
- **나노디그리 이수 조건** : 수행과제 + 파이프라인/에이전트 산출물 + 테스트 리포트.

## 5. Stage 3 — 학습 계획 (모듈)
- **M1. 테스트 디자인 & 하네스** — 전략·레벨·환경, 어떤 기준으로. *(시연)*
- **M2. CT 피드백 루프** — 실패→재시도→수정, 제품·피처 레벨 파이프라인. *(적용)*
- **M3. 게이팅 자동화 에이전트** — 산출물 게이트 평가 에이전트 구축. *(적용)*
- **M4. 고비용 검증의 대중화** — 뮤테이션·구조 테스트, 표준 정합. *(통합)*

## 6. 도구 & 자원
- **도구** : CI/CD(파이프라인), 뮤테이션 테스트 도구, Selenium/Playwright + Agentic Testing, 게이팅 에이전트.
- **자원** : Agentic Testing 가이드, Fowler(sensors/mutation), ISO 29119.
