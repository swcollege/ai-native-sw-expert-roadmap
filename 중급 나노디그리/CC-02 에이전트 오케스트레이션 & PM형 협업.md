## 0. 과정 한 줄 정의
개발자가 **에이전트를 조율(orchestrate)하고 그 산출물을 판단·책임지는 'PM형' 역량**과, 흩어진 작업을 **E2E로 연결**하는 멀티에이전트 워크플로우 설계 역량을 기른다.

## 1. 대상 학습자 & 진입 역량
- **대상** : 초급 이수자 전원(전 트랙).
- **진입 역량** : CC-01(컨텍스트·하네스) 동시/선행 수강 권장, 단일 에이전트 사용 경험.
- **비대상** : 없음(공통 선수).

## 2. 교수설계 근거
- **적용 방법론** : Backward Design · Bloom · Gagné 9 Events · Merrill(과제중심) · CBE.
- **설계 데이터 근거**
  - (내부) "사람은 에이전트를 굴리는 인원 / **최종적으로 수갑 찰(책임질) 사람** → 개발자 = 에이전트 조율자 + 산출물 판단자(**PM형**)"(거버넌스 책임(D)) / "워크플로우가 다 연결돼 있지도 않고 **개별로 뿔뿔이 흩어져 있다**"(PO 리더(A)) / "E2E로 연결하려면 **오케스트레이션을 누가** 할 것이냐"(거버넌스 책임(D)).
  - (외부) Anthropic **3-에이전트 하네스(계획·생성·평가)** / "Agent = Model + Harness" / arXiv C5(오케스트레이션)·C8(Human–AI 협업).
- **왜 공통인가** : PM형 조율·판단은 특정 전문가가 아닌 **전 개발자의 '일하는 방식' 전환**(Execution→AI / Decision→Human).

## 3. Stage 1 — 기대 성취
- **핵심 질문** : 한 명이 여러 에이전트를 어떻게 분해·배치·조율하는가? "끝났다"를 누가 어떻게 판단하는가?
- **전이 목표** : 현업 과제를 작업 분해→멀티에이전트 배치→E2E 연결→산출물 게이트로 **1인이 운영**한다.
- **이해** : ① 분해(decomposition)가 오케스트레이션의 8할 ② 에이전트는 전권 위임이 아니라 '자율성 슬라이더' ③ **최종 책임의 도장은 사람**.
- **학습목표(Bloom)**
  - LO1 (적용) 과제를 계획/생성/평가 역할의 에이전트로 분해·구성한다.
  - LO2 (분석) 단계 간 산출물 인터페이스(핸드오프) **단절 지점**을 식별한다.
  - LO3 (평가) AI 산출물을 기준에 비춰 **채택/반려(게이트)**한다.
  - LO4 (창안) 흩어진 작업을 **E2E 워크플로우**로 연결한다.

## 4. Stage 2 — 평가 증거 / DoD
- **수행과제** : 현업 미니 워크플로우를 **3+ 에이전트(계획·생성·평가)로 분해·연결**하고, 산출물 게이트(채택 기준)와 핸드오프를 설계·시연.
- **루브릭** : ① 분해 적절성 ② 핸드오프 무결성 ③ 게이트(판단) 명확성 ④ E2E 동작/재현.
- **이수 DoD** : E2E 워크플로우가 사람 개입 최소로 1회 완주 + **게이트에서 불량 산출물 1건 이상 반려** 시연.
- **나노디그리 이수 조건** : 수행과제 + 워크플로우 다이어그램·실행 로그 제출.

## 5. Stage 3 — 학습 계획 (모듈)
- **M1. PM형 마인드셋** — Execution→AI / Decision→Human, 산출물 판단·책임, "수갑 찰 사람". *(주의환기·동기)*
- **M2. 작업 분해 & 멀티에이전트** — 계획/생성/평가 역할 분담(Anthropic 패턴), 역할 정의. *(시연→적용)*
- **M3. E2E 연결 & 핸드오프** — 단계 간 인터페이스, 단절 지점 진단, 게이트 배치. *(적용)*
- **M4. 통합 프로젝트** — 현업 워크플로우 오케스트레이션. *(통합)*
