중급 나노디그리 과정 (Intermediate Nano-Degrees)

공통 2 + 트랙별 2×3 · 교수설계 = Backward Design·Bloom·Gagné·Merrill·CBE
전문가 인증 도전 자격 = 공통 2과정(CC-01·CC-02) + 해당 트랙 2과정 이수  ·  카드 클릭 → 우측 패널 상세 / MD ↗ 로 과정 설계서(교수설계 문서) 열람

Harness 논점 — SW College 최종 판단

"공통 역량(개인화기) + 도메인결합 전문기능(시스템 설계) + 거버넌스/운영 조직 — 단, 셋 다 '독립 전문가 직군'은 아니다."

① 공통 역량 층

컨텍스트·토큰효율(커맨드化)·스킬·피드백 루프는 모든 개발자의 기본기. → 중급 공통 나노디그리(CC-01·02).
Fowler "practice, not a role"

② 전문 기능 층

E2E 오케스트레이션·게이팅·IDP는 전담 전문성을 요하나 Arch/SDET/Security 도메인 위에 얹히는 능력. → 전문가 셀의 현업과제(하네스).

③ 거버넌스 조직

토큰·예산·중복제거·교육 담당 운영 조직은 필요하되 인증 직군 아님 · 과도기적(성숙 시 축소).
Nestr "↔ governance"
